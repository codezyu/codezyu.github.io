计算机组成原理--硬件

Posted on 2021-01-31 Edited on 2021-03-09 In 编程基础

0.微型计算机

Micro Computer

IC 元件外露的、用于控制的计算机

1.CPU

central processing unit

提取，解码，执行

CPU从RAM提取指令，然后解码该指令内容，最后执行。

所有的数据处理都在CPU中

(RAM:主存，随机存取存储器，为临时数据存储介质)

CPU与其它部件的运行需要时钟信号

就是由内含晶振的、 被称作时钟发生器的元件发出的电信号。

晶振：一种利用石英晶体（ 又称水晶） 的压电效应产生高精度振荡频率的电子元件

¶Z80 CPU

一种8比特的CPU,CPU 上数据总线的条数， 或者 CPU 内部参与运算的寄存器的容量， 都可以作为衡量 CPU 性能的比特数。 在Z80 CPU 中， 无论是数据总线的条数还是寄存器的容量都是 8 比特。

¶构成

¶控制单元(CU,control unit)：

从内存提取指令并解码运算，协调参与机器指令执行的步骤序列

¶寄存器

暂存指令，数据，地址

位于 CPU 和 I/O 中的数据存储器

¶标志寄存器

Flags Register

用于在运算指令执行后，存储运算结果的某些状态

一旦执行了算术运算、 逻辑运算、 比较运算等指令后， 标志寄存器并不会存放运算结果的值， 而是会把运算后的某些状态存储起来，例如运算结果是否为 0、 是否产生了负数、 是否有溢出（Overflow） ，发生了进位，数字大小的比较等。

¶累加器

累加器一般是运算的核心，连接的导线更多

¶PC（ Program Counter， 程序计数器 ） 程序指针

存储着指向 CPU 接下来要执行的指令的地址 (指令存储都在内存中)

例如执行完10号地址开始的三字节指令后

PC寄存器的值就会变成10+3=13（顺序执行）

会随着时钟信号自动更新，每解释执行完一条指令， PC 寄存器的
值就会自动被更新为下一条指令的地址

可以说程序就是依靠不断变化的 PC 寄存器的值运行起来的

¶程序的流程

顺序执行

按照指令记录在内存中的先后顺序依次执行的一种流程。

条件分支

根据若干个条件的成立与否， 在程序的流程中产生若干个分支的一种流程

循环

在程序的特定范围内反复执行若干次的一种流程。

使用了机器语言或汇编语言所提供的跳转指令， 就可以将PC 寄存器的值设置为任意的内存地址。 如果将它的值设为之前执行过的步骤所对应的内存地址， 那么就构成了循环 .通过条件分支返回到之前处理过的指令来实现循环的, 跳转到之后尚未处理的步骤就是条件分支

中断处理

程序的流程突然跳转到程序中的特定地方，这 样 的 地 方 被 称 为 中 断 处 理 例 程（ Routine） 或 是 中 断 处 理 程 序（ Handler），

使正常的流程中断， 中断后再返回到之前流程的过程就是中断处理流程

中断请求信号一般由I/O模块

¶SP寄存器

也叫作“栈顶指针”

用于在内存中创建出一块称为“栈” 的临时数据存储区域

¶控制器

将内存中指令，数据读入寄存器，并根据指令控制计算机

¶算数逻辑单元(ALU,arithmetic logic unit)

处理算数和逻辑运算(AND OR NOT)

¶运算器

运算从内存读入寄存器的数据

¶时钟(clock)

对CPU内部操作与系统其它组件进行同步

每隔一定时间就变换一次电压的高低

输出时钟信号的元件叫作“时钟发生器”

时钟信号的频率可以衡量 CPU 的运转速度

从功能来看，可分为由电信号联通的四部分

内存–>寄存器

¶引脚

¶地址总线引脚

引脚位置表示二进制数字中的位置，进而有$2^{n}$个，每个引脚连接一个数据存取单元(内存存储单元或I/O单元)

¶数据总线引脚

仍然表示二进制各个数字的位置，每个引脚代表一个bit的数据

数据总线引脚数表示一次性处理的数据大小(bit)

如大于这个，需要进行切分(如8位)

在确定存取数据地址后

¶控制引脚

这些引脚上输入输出的电信号具有控制 IC 的功能

总线

传输数据，总线是连接到 CPU 中数据引脚、 地址引脚、 控制引脚上的电路的统称

计算机部件之间也可以分散连接

信息传送分类

并行

串行

¶片内总线

芯片内部的总线

所以对寄存器的运算速度很快

¶系统总线

计算机各部件之间的信息传输线

¶地址总线

保存指令和数据的地址

单向 与存储地址、 I/O地址有关

由CPU输出数据地址

取数据时，CPU将数据所在的存储位置送到地址线上

地址线的位数和存储单元个数有关

地址线为20根，则存储单元数为$2^{20}$(每个引脚表示一bit)

¶控制总线

数据，地址总线为各部件共享，控制总线确定哪一个获得使用权

控制总线是个总称，有多少根，代表CPU对外部器件由多少种控制

控制总线的宽度代表CPU对外部器件的控制能力

二进制信号对所有连接在系统总线的设备行为进行同步

时钟与 CPU 和系统总线相关的每一个操作都是由一个恒定速率的内部时钟脉冲来进行同步。机器指令的基本时间单位是机器周期 (machine cycle) 或时钟周期 (clock cycle)。

一个时钟周期的时长是一个完整时钟脉冲所需要的时间

执行一条机器指令最少需要 1 个时钟周期

¶控制信号

中断请求、总线请求

存储器读、存储器写

总线允许、中断确认

¶数据总线

各功能部件之间的数据信息

CPU一次取得的大小和数据总线宽度（有多少根）有关

双向 与机器字长、存储字长有关

¶I/O总线

CPU和IO设备中传输数据

以CPU为主时的总线

¶通信总线

用于 计算机系统之间 或 计算机系统与其他系统（如控制仪表、移动通信等）之间的通信

并行通信总线

串行通信总线

¶指标

总线宽度

数据总线 的根数

标准传输率

每秒传输的最大字节数（MBps）

时钟同步/异步

异步会慢一点，异步：与时钟不同步工作

总线复用

8086（数据线：16，地址线：20）后广泛使用，地址线 与 数据线 复用（一物多用，一个管脚既可以表示地址信号也可以表示数据信号，数据和地址信号因为不会同时出现，一般地址信号在前，数据信号在后（按址访问））

信号线数

地址线、数据线和控制线的 总和

总线控制方式

并发、自动、仲裁、逻辑、计数

负载能力

¶总线标准

有许多

RS-232

USB(扩展能力很强)

¶总线结构

¶单总线结构

总线被各个部件都使用

¶多总线结构

I/O设备速度较慢，因此从总线分离出来

¶双总线结构

I/O通过通道（特殊处理器，由CPU进行控制）连接总线

¶三总线结构

Cache出现后加装局部总线

¶四总线结构

增加了高速总线，用于外设

¶总线控制（重点）

总线来连接多个部件，需要确定发送信息

¶总线判优控制

主设备：对总线有控制权，总线信息传送由主设备启动

从设备：响应主设备发来的总线命令

主设备同时要使用总线，由总线控制器进行判断谁获取总线控制权

¶集中式

控制逻辑集中在一处(如CPU)

¶优先权仲裁方式

BS:总线忙

BR:总线请求

BG:总线同意

¶链式查询

BG串行经过接口，BG到达的接口若有总线请求，BG信号不再下传

建立BS信号（占用了总线）

链式查询中，总线控制部件最近优先级最高

优点：简易，容易扩充设备，

缺点

总线控制部件不能直接确定是哪一台外设发出的请求信号，需要一个个去询问

电路故障影响大，优先级低的设备很难获得请求

不是按照事情轻重缓急进行处理

¶计数器定时查询

多了设备地址线，少了BG，多了计数器（总线控制部件中）

I/O接口从0（设备地址号）开始增加编号

1.BR线传来总线请求信号

判断条件：总线未被使用(BS=0)

2.计数器开始计数，通过设备地址线发出一组地址信号

3.请求占用总线的设备地址和计数值一致，获得总线使用权

4.终止计数查询

特点

计数从0开始：

优先级按照0~n降序排列

计数器可以按照上次计数终止点开始，循环计数：

优先级相等

初始值由程序设定：

优先级可以改变

设备地址线：$log_{2}(n)$(n为设备数，二进制表示n,每一条代表一位)

¶独立请求方式

每台设备都有总线请求BR和总线同意BG

总线控制部件有排队电路，通过优先次序确定响应哪台设备

特点：

响应速度快，可通过程序控制优先级

但控制线比较多，总线控制更为复杂

¶分布式

逻辑控制分散在与总线连接的各个部件或设备

内存存储单元

计算机所处理的数据都存储在了被称为内存的 IC（ Integrated
Circuit， 集成电路） 中。 在一般的个人计算机中， 内存内部被分割成了若干个数据存储单元， 每个单元可以存储 8 比特的数据（ 8 比特 = 1 字节）。

每个存储单元从0开始顺序编号

指令和数据没有区别

高级语言的变量即是把一块内存空间进行命名

在程序运行时保存指令与数据

接受来自CPU的数据请求，将数据从RAM传输到CPU,并从CPU传输到内存

反汇编：机器语言转化为汇编语言（为什么是汇编语言呢？因为机器语言的指令和汇编语言的指令是一一对应的）

标识内存或I/O存储单元的数字称作地址

¶随机存储器(RAM)

可读可写，必须带电存储

¶只读存储器(ROM)

只读

关机内容不丢失

¶CPU对存储器的读写

CPU读写需要和外部器件进行交互

1.存储单元的地址（地址线）

2.对什么器件进行操作（读还是写）（控制线）

3.数据信息（数据线）

eg:

CPU通过地址线将地址信息发出

CPU通过控制线对内存发出读命令，选中存储器芯片，通知它

存储器将对应地址的数据通过数据线送到CPU

¶存储顺序

在将一个 2 字节的数据存储到内存时
存储顺序是低 8 位在前、 高 8 位在后（ 也就是逆序存储）。 这样的存储顺序叫作“小端序”（ Little Endian）， 与此相反， 将数据由高位到低位
顺序地存储到内存的存储顺序则叫作“大端序”（ Big Endian）。 根据
CPU 种类的不同， 有的 CPU 使用大端序， 有的 CPU 使用小端序

¶引脚

和CPU一样

需要和CPU对应的引脚相连
