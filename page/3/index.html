计算机组成原理--硬件

Posted on 2021-01-31 Edited on 2021-03-09 In 编程基础

0.微型计算机

Micro Computer

IC 元件外露的、用于控制的计算机

1.CPU

central processing unit

提取，解码，执行

CPU从RAM提取指令，然后解码该指令内容，最后执行。

所有的数据处理都在CPU中

(RAM:主存，随机存取存储器，为临时数据存储介质)

CPU与其它部件的运行需要时钟信号

就是由内含晶振的、 被称作时钟发生器的元件发出的电信号。

晶振：一种利用石英晶体（ 又称水晶） 的压电效应产生高精度振荡频率的电子元件

Z80 CPU

一种8比特的CPU,CPU 上数据总线的条数， 或者 CPU 内部参与运算的寄存器的容量， 都可以作为衡量 CPU 性能的比特数。 在Z80 CPU 中， 无论是数据总线的条数还是寄存器的容量都是 8 比特。

构成

控制单元(CU,control unit)：

从内存提取指令并解码运算，协调参与机器指令执行的步骤序列

寄存器

暂存指令，数据，地址

位于 CPU 和 I/O 中的数据存储器

标志寄存器

Flags Register

用于在运算指令执行后，存储运算结果的某些状态

一旦执行了算术运算、 逻辑运算、 比较运算等指令后， 标志寄存器并不会存放运算结果的值， 而是会把运算后的某些状态存储起来，例如运算结果是否为 0、 是否产生了负数、 是否有溢出（Overflow） ，发生了进位，数字大小的比较等。

累加器

累加器一般是运算的核心，连接的导线更多

PC（ Program Counter， 程序计数器 ） 程序指针

存储着指向 CPU 接下来要执行的指令的地址 (指令存储都在内存中)

例如执行完10号地址开始的三字节指令后

PC寄存器的值就会变成10+3=13（顺序执行）

会随着时钟信号自动更新，每解释执行完一条指令， PC 寄存器的
值就会自动被更新为下一条指令的地址

可以说程序就是依靠不断变化的 PC 寄存器的值运行起来的

程序的流程

顺序执行

按照指令记录在内存中的先后顺序依次执行的一种流程。

条件分支

根据若干个条件的成立与否， 在程序的流程中产生若干个分支的一种流程

循环

在程序的特定范围内反复执行若干次的一种流程。

使用了机器语言或汇编语言所提供的跳转指令， 就可以将PC 寄存器的值设置为任意的内存地址。 如果将它的值设为之前执行过的步骤所对应的内存地址， 那么就构成了循环 .通过条件分支返回到之前处理过的指令来实现循环的, 跳转到之后尚未处理的步骤就是条件分支

中断处理

程序的流程突然跳转到程序中的特定地方，这 样 的 地 方 被 称 为 中 断 处 理 例 程（ Routine） 或 是 中 断 处 理 程 序（ Handler），

使正常的流程中断， 中断后再返回到之前流程的过程就是中断处理流程

中断请求信号一般由I/O模块

SP寄存器

也叫作“栈顶指针”

用于在内存中创建出一块称为“栈” 的临时数据存储区域

控制器

将内存中指令，数据读入寄存器，并根据指令控制计算机

算数逻辑单元(ALU,arithmetic logic unit)

处理算数和逻辑运算(AND OR NOT)

运算器

运算从内存读入寄存器的数据

时钟(clock)

对CPU内部操作与系统其它组件进行同步

每隔一定时间就变换一次电压的高低

输出时钟信号的元件叫作“时钟发生器”

时钟信号的频率可以衡量 CPU 的运转速度

SPFA算法

Posted on 2021-01-31 In 算法

0.简介—最短路问题

BF（bellman-Ford ）算法：适合有向图,不是字符串的暴力匹配算法。

负权边：权重为负数的边

负权环：权重和为负数的环

用于计算起点到各个节点的最短距离，相比Dijkstra,允许有负权边（Dijkstra算法不允许负权边的出现）。但是复杂度为O(VE)

同时也可以检测负权环

步骤：

对于不存在负权环的图，求解最短路径：

利用已经找到的最短路径更新其它点的dist[]

顶点数：n

边数：m

源点：s

数组dist[i]记录从源点到顶点i的最短路径

dist[s]初始化为0,其余初始化为MAX

循环该操作n-1次:

对于边(u,v),权值为w(u,,v)

如果dist[u]+w(u,v)<dist[v]

则让dist[v]=dist[u]+w(u,v)

对比

我们先来看看Dijsktra算法

同样初始化

选取距离最近的点作为起始点即u

循环该操作n-1次

对于边(u,v),权值为w(u,,v)

如果dist[u]+w(u,v)<dist[v]

则让dist[v]=dist[u]+w(u,v)

Dijsktra算法本质是局部贪心

其实和BF算法看起来区别不大

检测负权环

在求解最短路径的n-1次循环基础上，再进行第n次循环

若存在

dist[u]+w(u,v)<dist[v]

则该图存在负权回路

而SPFA在此基础上进行了简单的优化

平均复杂度为O(KM)

SPFA可以求最短路，还可求负权边，负环，判环问题

HTTP协议

HyperText Transfer Protocol 超文本传输协议

指双方间传输超文本数据的规范

客户端向服务器发送请求消息（包括URI4和方法5，同时还包括附加信息的头字段，但是头字段可以省略）

服务器收到请求消息会进行解析，并且把结果放在响应消息中，响应消息的开头会有状态码，状态码含义如下

组成

1.超文本：文字，图片，视频，超链接（能够从一个超文本跳转到另一个超文本）的混合体

2.传输：HTTP为双向协议，数据可以相互传输

3.协议：计算机之间交流的规范，以及相关控制和错误处理方式

状态码
含义 状态码
1xx 提示信息，当前为协议处理的中间状态
2xx 成功，报文被收到并且被正确处理 200(非head请求，返回的响应头部有body数据)，204（响应头没有body数据），206（用于分块，或者端点续传，返回的body数据只是其中的一部分）
3xx 重定向，资源位置发生变动，需要客户端重新发送请求 301（永久重定向，请求的资源已经不在了），302（临时重定向，请求的资源还在），304
4xx 客户端错误，请求报文有误，服务器无法处理 400，403，404
5xx 服务器错误，服务器在请求时内部发生了错误 500,501,502,503
