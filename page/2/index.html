模式（ Schema）

外模式（ External Schema）

也称子模式或用户模式

外模式的地位：介于模式与应用之间

模式与外模式的关系

一对多

外模式通常是模式的子集
一个数据库可以有多个外模式。反映了不同的用户的应用需求、
看待数据的方式、对数据保密的要求
对模式中同一数据，在外模式中的结构、类型、长度、保密级别等都可以不同

外模式与应用的关系

一对多
同一外模式也可以为某一用户的多个应用系统所使用
但一个应用程序只能使用一个外模式

外模式的用途

保证数据库安全性的一个有力措施
每个用户只能看见和访问所对应的外模式中的数据

1.用户账户控制(UAC)策略

现代操作系统为了数据安全性设计的，即非root用户不能使用系统文件等等

一般说来C上的软件需要管理员权限

在appdata下，这是用户可使用的文件夹，可以无需管理员权限

管理员权限在win下拥有最高的权限

UAC下，管理员用户直接双击软件，则软件获得的权限为降权处理。

而有的软件标记为提权才能运行，所以这个时候就会跳出用户账户控制这个询问窗口了，一般来说安装软件等等都会跳出这个。

当然软件可以主动申请提权,但是需要重新申请进程

从目的上看，在一些恶意程序上没有实现保护的作用。

1.读写文件，截屏，连接网络，模拟鼠标键盘不需要管理员权限

系统仅仅保证两个账户资源尽量不共通

某些勒索病毒会调用vssadmin来删除win的备份，此时UAC具有部分作用

但是WannaCry勒索病毒在内核中，自带最高权限

软件安装后门，或者木马，UAC并不会干涉

2.不能制止弹窗，自启，垃圾文件制造等流氓行为

系统对于每个用户提供了独立的配置，包括自启，管理员权限下，可以更改全局设置，影响所有用户。

3.恶意软件拿到管理员权限会静默拿到管理员权限

1.奇异值分解(SVD: singular value decomposition)

机器学习中广泛应用的算法

1.特征值和特征向量

A是一个$n*n$的实对称矩阵，

$x$是一个$n$维向量，

则

$\lambda$是矩阵A的特征值

而$x$是矩阵$A$的特征值$\lambda$所对应的特征向量。

特征值和特征向量的作用

若$A$的所有特征值对应的n个特征向量线性无关，那么

$A$=$W\Sigma W^{-1}$

$W$是这些特征向量所张成的矩阵$n \times n$ 维矩阵，而$\Sigma$为这n个特征值为主对角线的$n \times n$维矩阵

这样$A$的进行幂运算会有很大好处，当然也会有标准化。这里暂时不谈

但问题是这个矩阵A必须为方阵

SVD

当$A$不是方阵时，我们定义$A$的SVD为

$A$=$U\Sigma V^{T}$

A为$ m \times n$的矩阵

U为$ m \times m$的矩阵

$\Sigma$为$ m \times n$的矩阵

$V$为$n \times n$的矩阵

为了得到这三个矩阵

我们首先得到$A^{T}$$A$而这是一个n阶方阵

对于n阶方阵，我们可以进行特征分解得到特征值和特征向量

$(A^{T}A)$$v_{i}$=$\lambda_{i} v_{i}$

1.线性回归算法

training set + algorithm—->hypothesis 本人仍然将其理解为模型

test set - hypothesis —->estimated result

代价函数

为模型选择合适的参数

建模误差：预测值和实际值的差距

线性回归中一般为平方误差函数

通过梯度下降求出代价函数最小值

损失函数，代价函数(loss function)

用于度量拟合程度，损失函数越小，模型拟合得越好

风险函数(risk function)

是损失函数的期望，由于输入的(X,Y)遵循联合分布，但是联合分布未知，无法计算，我们的历史数据训练集$f(X)$关于训练集的平均损失成为经验风险(empirical risk),即$\frac{1}{N}\sum_{i=1}^{N}L(y_{i},f(x_{i}))$,我们的目标是最小化经验风险，称为经验风险最小化

但是，如果模型拟合得过好，反而会过度学习历史数据，真正预测的效果往往不好，这种情况就是过拟合

一般来说，造成这样是因为其函数过于复杂，拟合因此很精准。

多变量线性回归

$h_{\theta}=\theta_{0}x_{0}+\theta_{1}x_{1}+\cdots+\theta_{n}x_{n}$

参数为$n+1$维的向量，因此训练实例也为$n+1$维的向量，特征方程的维度为$m*(n+1$)维。

一个训练实例的特征:{${x_{0},x_{1},\cdots,x_{n}}$}

特征方程：m个特征向量($n+1$)，所以为$m*(n+1)$维度

所以可以简化为：

$h_{\theta}(x)=\theta^{T}X$

这样便获得参数矩阵

多变量线性回归的代价函数

代价函数为所有建模误差的平方和

随机选择一系列的参数值，计算所有的预测结果后，在给所有的参数一个新的值，如此循环知道收敛

梯度下降

特征缩放—-标准化

面对多维特征问题时，我们要保证特征具有相近的尺度，梯度下降能够更快收敛。

对于不同特征的尺度，如果不一致，会导致在一个维度上变化的幅度不能很大。

因此我们要尝试把所有尺度缩放到-1和1之间

$x_{n}=\frac{x_{n}-u_{n}}{s_{n}}$

$u_{n}$是数据的平均值，$s_{n}$是标准差

1.问题

在知乎看到的一个有趣的问题

若非负整数a,b，其中a<b,a经过不限顺序和次数的+1和x2运算变成b，问最少的步数是多少

关于这个问题，由许多很好的方法

解法一

其中最简单易懂而且很高效的做法是pecoo老师（其实是我们系的大佬）

考虑a变成b所需步骤

我们不妨从最后一步开始入手

对于目标b

1.如果b是奇数，那么最后一步必然是+1操作

2.如果b是偶数，要实现最少的步数，我们要尽量选择x2操作，但如果$b/2<a$,那a只能经过+1操作变成b,那么最小步数就是b-a

​ 如果$b/2>=a$

​ $(a+x)\times 2$和$（a\times 2)+y$相比，明显前面所需的步数少，即在x2的作用下，+1的操作更有效

​ 因此我们将$b/2$,再进入递归判断

解法二

思路奇特，可以看出这位答主Muchan编程基础相当强

x2相当于在二进制后加0

+1相当于末尾加1

为了最少的步骤，先将a进行+1操作将a变成b的前缀

对于b剩下的每一位，为0则x2,为1则x2再加1(可把1看成0+1，为0表示一位，需要x2进位，+1需要该位变为1)

a:1(2) b:101(2)

所以a的步骤为x2,x2+1

那么如何将a变成b的前缀呢

很简单，将a不断+1变成前缀即可（x2会加长a的二进制长度）

要求：b的前缀一定要大于等于a

a:1(2) b:101(2)

二进制a长度为1，对应b的前缀为1，刚好匹配

a:11(2) b:1001(2)

二进制a的长度为2，对应b的前缀为10，小于a,所以对a,b进行处理使得前缀相等

首先b的前缀为100，肯定大于11，那么a只需进行+1（进行x2操作一定会继续大于目前b的前缀，因为b前缀的前缀10小于11）

所以11+1=100
